What changed from the first mockup

Now it's layout A only, and each side is the existing AlignmentPanel verbatim, top to bottom: the UHB source strip, then the word inventory strip (your real "word bank" — every word in stream order, aligned ones greyed, unaligned ones live & draggable, with the colors / hover-link toggles and the resize grip), then Groups · n, the cards, and the action bar with its own Save. No bottom word bank. The only new chrome is the popup frame and the small shared reading strip up top.

Both editable

Each panel saves its own version independently — ULT and UST are different bible_versions, so their outbox PATCHes don't collide (different If-Match targets). Two action bars, two Save buttons.

The cross-panel hover (whole-group, on by default)

Hovering a Hebrew word rings its exact twin in the other panel (blue), and rings that word's whole group in each panel (gold — the English it maps to, plus any sibling Hebrew the group combines). Hover וָאַ֗עַן: ULT lights only "And I answered"; UST lights וָאַ֗עַן וָאֹמַ֔ר → "I asked" — the split-vs-combine, instantly. Governed by the existing per-panel hover-link toggle; lifting that hover state to the popup is the one real code change.

Edit the English inline (reading strip)

The top reading strip is editable — click into either side and change the wording without leaving the aligner (try it). In the real app this routes through the existing smart-edit path (replace.ts / alignment.ts): when a word-count-preserving change is made, alignments survive; otherwise the touched words drop back to the inventory as unaligned, exactly like editing in a normal scripture column. The edit saves the target verse via the same outbox PATCH, then the inventory and cards below re-derive. So: tweak the English up top, re-align the chips below — one view.

Settled

• One shared UHB strip across the top (both bibles align to the same Hebrew) — collapsible via the ▾.  • Reading strip kept.  • Two per-panel Save buttons (independent saves).  • No theme toggle — the popup follows the app/environment theme (this mockup respects prefers-color-scheme).
